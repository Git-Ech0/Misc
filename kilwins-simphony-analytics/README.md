# Kilwins Simphony Analytics

A Chrome (Manifest V3) extension that turns any Oracle Simphony / Reporting & Analytics
report into actionable, plain-English store-management recommendations.

It is targeted at the Kilwins reporting portal at
`https://simphony-home.mtu9.oraclerestaurants.com/portal/...`, but the table /
data-grid extractor is generic enough to work on **any** Oracle JET-rendered
report (`<oj-table>`, `<oj-data-grid>`, ARIA grids, native HTML tables).

## What it does

1. Adds a **floating "Analyze report" button** in the bottom-right corner of
   every Simphony portal page.
2. When you click it, the extension scrapes every table-like structure on the
   page (including same-origin iframes), parses cells into typed values
   (currency / number / percent / date / weekday / hour / string), and stores
   the snapshot locally in the browser.
3. Opens a **custom dashboard tab** that, for every captured table, generates
   a stack of recommendation cards:
   - **Sell more of these** (top movers by sales / revenue)
   - **Make less of these** (slow movers — cut prep to reduce waste)
   - **Buy more inventory of these** (high-velocity items)
   - **Promote these — high value, under-selling** (above-average revenue per
     unit but below-average volume)
   - **Put these on sale / discount** (bottom-quartile movers)
   - **Margin alerts** (lowest profit when both revenue and cost are present)
   - **Discount / void watchlist** (highest discount or comp totals)
   - **Staff up / cut hours** (busiest and slowest weekdays / hours / dates)
   - **Category breakdown** (where revenue actually came from)
   - **Snapshot** (totals, implied avg ticket, etc.)

Every card is generated by a heuristic rule that only fires when the relevant
columns are present in the report — so a *Sales by Weekday* report and a
*Menu Item Sales* report each surface a different (but appropriate) set of
recommendations.

## Install (unpacked)

1. Clone this repo (or just download the `kilwins-simphony-analytics/` folder).
2. Open `chrome://extensions` in Chrome.
3. Toggle **Developer mode** on (top-right).
4. Click **Load unpacked** and pick the `kilwins-simphony-analytics/` folder.
5. Visit a report on the Simphony portal — the floating brown button appears
   in the bottom-right corner.

## Tests

A small Node-only smoke test suite covers the parser + analyzer:

```bash
node --experimental-vm-modules tests/analyzer.test.mjs
```

It exercises:

- HTML `<table>` extraction
- Oracle JET-style ARIA `[role="grid"]` extraction
- Currency / percent / paren-negative / weekday parsing
- Each recommendation rule against a synthetic *Sales by Weekday* and
  *Menu Item Sales* report shape

## Versatility

The extractor never assumes a specific column order or report name. It:

- Reads headers from `<thead>`, the first row of `<th>` cells, or the first
  row if all else fails.
- Falls back to ARIA `role="row"` / `role="columnheader"` / `role="gridcell"`
  for `<oj-data-grid>` and other ARIA-only grids.
- Walks same-origin iframes in case the report is rendered inside an embed.
- Classifies columns by name pattern (revenue / cost / quantity / discount /
  weekday / hour / date / item) so the same rules drive any new Simphony
  report you point it at.

## Privacy

All scraping and analysis happens locally in the browser. Snapshots are written
to `chrome.storage.local` and never leave the device.
